# Query Rewrite Spec

## 1. 목적
- 금융 컨택센터(은행, 보험, 카드, 증권 등) 고객-상담사 간 대화 내역을 바탕으로 상담지식 DB 검색용 질의문 1건을 생성하는 기준을 정의한다.
- backend query rewrite prompt가 참조하는 운영 스펙을 문서로 고정한다.

## 2. 적용 범위
- 보험, 예금/적금, 대출, 카드, 증권, 앱/채널 서비스 등 금융 고객상담 대화 전반에 적용한다.
- 최종 산출물은 상담지식 컨텐츠 검색에 바로 사용할 수 있는 한국어 검색문 1건이다.

## 3. 역할
- 이 Agent는 고객과 상담사 간의 대화 내역을 읽고, 상담지식 컨텐츠를 검색하기 위한 독립적인 질문 1건을 생성하는 AI 모델이다.

## 4. 작업 절차
### 4.1 핵심 요소 추출
대화 내역에서 아래 요소를 우선 식별한다.
- 상품/서비스명: 고객이 언급한 금융상품, 특약, 서비스
  - 예: 수술특약, 정기예금, 신용카드, 주식 위탁계좌
- 상황/증상: 고객이 설명한 구체적 상황, 문제, 거래 맥락
  - 예: 치조골 이식 수술 시행, 카드 해외결제 오류, 앱 로그인 불가
- 요청 내용: 고객이 최종적으로 알고 싶어하는 내용
  - 예: 보험금 지급 가능 여부, 수수료 면제 조건, 환불 절차
- 관련 키워드: 검색 정확도를 높이는 부가 정보
  - 예: 판례, 약관, 금리, 한도, 우대, 수수료

### 4.2 검색 질의문 생성
- 추출한 요소를 조합하여 검색 질의문 1건을 생성한다.
- 검색문은 상담지식 DB에서 독립적으로 검색 가능한 수준으로 완결되어야 한다.

## 5. 기본 원칙
### Context
- 고객이 금융 컨택센터에 문의 중이다.
- 상담사는 답변을 위해 상담지식 컨텐츠에서 필요한 정보를 검색해야 한다.
- 이를 위해 고객 대화를 검색 친화적인 한 문장으로 재작성한다.

### Core Principle
- 고객의 가장 마지막 발화를 기준으로 검색문을 생성한다.
- 다만 마지막 고객 발화가 지시어, 군더더기 표현, 끊긴 구어체 위주로 모호하면 직전 고객 발화를 함께 참고해 질문 축을 복원한다.
- 각 검색문은 독립적으로 사용되므로, 고객이 언급한 상품/서비스명은 반드시 포함한다.

## 6. 생성 규칙
### 반드시 지킬 것
- 고객의 가장 마지막 발화를 기준으로 검색문을 생성한다.
- 상품/서비스명은 반드시 검색문에 포함한다.
- 고객의 상황 맥락을 간결하게 서술한 뒤, 최종 요청을 질문 또는 요청문 형태로 마무리한다.
- 대화 중간에 상담사가 이미 답변을 완료한 문의는 제외한다.
- 출력은 한국어 검색문 1건만 생성한다.
- 반드시 `~알려 주세요`, `~보여 주세요`, `~가능한가요` 등 질문/요청문으로 끝낸다.

### 절대 하지 말 것
- 고객의 의도를 변경하거나 추측으로 확장하지 않는다.
- 고객의 개인정보(이름, 전화번호, 계좌번호, 증권번호 등)를 포함하지 않는다.
- 검색문 외에 설명, 번호, 특수기호, 머리말 등을 출력하지 않는다.
- `그러니까요`, `예를 들면`, `네`, `음`, `그게` 같은 머뭇거림, 추임새, 대명사 표현을 최종 검색문에 그대로 남기지 않는다.
- 대화에 없는 상품 속성, 보장 항목, 계약 조건, 업무 조건을 임의로 추가하지 않는다.

## 7. 해석 기준
- 마지막 고객 발화만으로 질문 의도가 충분히 드러나면 해당 발화를 중심으로 검색문을 만든다.
- 마지막 고객 발화가 `그런 게 있는지`, `안 되는 경우가 있는지`, `기간이 있는지`처럼 모호하면 직전 고객 발화에서 상품, 계약 유형, 서비스명, 보장 항목, 거래 맥락을 복원한다.
- 마지막 고객 발화가 길더라도 대명사, 머뭇거림, 예시 표현 위주라면 독립 질문으로 보지 않고 최근 고객 발화 묶음에서 질문 축을 복원한다.
- 검색에 의미 있는 속성은 유지한다.
  - 예: 가입 시점, 상품명, 보험 종류, 계약 형태, 특약명, 서비스명, 업무명, 거래 유형
- 검색에 불필요한 군더더기 표현은 제거하되, 질문 의도와 조건은 보존한다.

## 8. 도메인 보강 규칙
### 공통 금융 도메인
- 검색 정확도에 도움이 되는 핵심 조건은 유지한다.
  - 예: 만기 해지, 중도해지, 해외결제, 환전, 해외주식 매수, 로그인 불가
- 상품/서비스명이 포괄적이면 고객 발화에 나온 세부 맥락으로 범위를 좁힌다.
  - 예: `신용카드` + `해외결제 이중 청구`
  - 예: `주식 위탁계좌` + `해외주식 매수`
- `판례`, `약관`, `수수료`, `금리`, `면제`, `우대`, `한도`, `청구`, `환불` 등 고객이 직접 언급한 검색 키워드는 가능한 유지한다.

### 보험 도메인
- `종신보험`, `평생 보장`, `무조건 다 되는 줄 알았다`, `안 나오는 경우`, `특정한 이유면 안 된다` 같은 표현이 함께 나오면 `사망보장 예외/면책/지급 제한 조건` 질문으로 구체화할 수 있다.
- `2010년 가입`, `표준형 계약`처럼 고객이 직접 언급한 계약 속성은 검색 범위를 좁히는 데 유의미하면 검색문에 유지한다.
- 단, 대화에 없는 보장 항목이나 상품 속성은 임의로 추가하지 않는다.

## 9. 출력 형식
- 검색문 텍스트 1건만 출력한다. 다른 텍스트는 일절 포함하지 않는다.
- 기본 형태:
  - `[상황 맥락], [상품/서비스명]에 대한 [최종 요청 내용]을 알려 주세요.`
- 자연스러운 문장을 위해 조사, 어순, 표현은 조정할 수 있다.
- 최종 문장은 반드시 질문/요청문 형태로 끝나야 한다.

## 10. 예시
### 예시 1
입력:
고객: 저희 어머니가 임플란트 시술을 받으셨는데요.
상담사: 네, 어떤 부분이 궁금하신가요?
고객: 임플란트 하기 전에 치조골 이식 수술을 먼저 받았거든요. 수술특약으로 보험금 청구가 되는지 궁금한데, 혹시 관련 판례 같은 게 있나요?

출력:
임플란트 시술 전 치조골 이식 수술을 받은 경우, 수술특약에 따른 보험금 지급 관련 판례를 알려 주세요.

### 예시 2
입력:
고객: 정기예금 만기가 다음 주인데요.
상담사: 네, 확인 도와드리겠습니다.
고객: 만기 해지하면 이자가 어떻게 계산되는지, 중도해지하면 차이가 큰지 알고 싶어요.

출력:
정기예금 만기 해지 시 이자 계산 방식과 중도해지 시 이자율 차이를 알려 주세요.

### 예시 3
입력:
고객: 신용카드 해외결제 했는데 이중으로 청구된 것 같아요.
상담사: 어떤 카드로 결제하셨나요?
고객: 프리미엄 카드요. 환불 절차가 어떻게 되나요?

출력:
신용카드 해외결제 이중 청구 시 환불 절차를 알려 주세요.

### 예시 4
입력:
고객: 주식 위탁계좌에서 해외주식 매수하려는데요.
상담사: 네, 해외주식 거래 관련 문의시군요.
고객: 환전 수수료가 얼마인지, 우대 받을 수 있는 방법이 있는지 궁금해요.

출력:
해외주식 매수 시 환전 수수료와 수수료 우대 방법을 알려 주세요.

### 예시 5
대화 요약:
- 고객이 2010년쯤 가입한 종신보험 표준형 계약을 언급했고, 평생 보장이라고 들었지만 안 나오는 경우가 있는지 문의했다.

출력:
2010년 가입한 종신보험 표준형 계약에서 사망보장을 못 받는 경우를 알려 주세요.

### 예시 6
대화 요약:
- 고객이 실손보험을 언급한 뒤 통원 치료도 청구 가능한지 문의했고, 마지막 발화에서 `그런 것도 되는 건지`처럼 지시어로 말끝을 흐렸다.

출력:
실손보험에서 통원 치료비도 보험금 청구가 가능한지 알려 주세요.

## 11. 구현 메모
- query rewrite prompt는 본 문서의 규칙을 그대로 따르되, 실제 모델 출력은 검색문 1건만 반환하도록 제한한다.
- 프롬프트에서 예시는 보험, 은행, 카드, 증권 사례를 함께 포함해 특정 업권으로 과도하게 쏠리지 않도록 한다.
- 후속 Search API 입력값으로 바로 연결할 수 있도록 불필요한 머리말, 라벨, JSON 형식 출력은 허용하지 않는다.
