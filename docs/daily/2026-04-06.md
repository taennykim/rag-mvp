## 3. 이슈 및 문제
- RAG 서버는 파일 내용 기준으로는 현재 서버와 맞췄지만, git commit 자체는 여전히 로컬/GitHub와 다르다.
- 따라서 당분간 RAG 서버 동기화 여부는 git HEAD보다 핵심 파일 해시 기준으로 판단하는 편이 안전하다.
- EC2 metadata option 변경은 요청 직후 `pending` 상태를 거칠 수 있으므로 다음 확인 시 `applied` 여부를 다시 봐야 한다.
- Terraform state와 실제 AWS 설정 사이에 drift가 생기지 않도록 이후 `terraform plan` 시 metadata option도 같이 확인해야 한다.
- 현재 인덱스에는 약관 PDF가 없고 duplicate `DOCX`가 포함되어 있어, 질문 세트 품질 판정 전 인덱스 구성을 다시 정리할 필요가 있다.
- `이 상품의 적용 이율은 얼마인가?`는 retrieval이 틀렸는데 answer가 맞아 보이는 상태라, 외부 근거 기반 답변과 내부 모델 추론을 같은 채널에서 섞으면 안 된다.
- 인덱스 정상화 후 `청약 철회` 질문은 약관 PDF로 회복됐지만, `적용 이율` 질문은 여전히 약관 쪽으로 쏠리는 케이스가 있어 `document_hint` 기반 retrieval 보정이 아직 부족하다.
- backend 재기동 시 `pkill`이 SSH 세션과 충돌하는 경우가 있어 운영 재기동 절차를 더 안전하게 정리할 필요가 있다.
