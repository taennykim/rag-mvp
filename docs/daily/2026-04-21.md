## 3. 이슈 및 문제
- custom Query Rewrite에서 `Qwen2.5-7B-Instruct` 404가 남았던 이유는 backend 기본 모델이 아니라, 사용자가 `query_rewrite_model=custom`으로 unsupported model name을 보낸 요청이 로그에 남았기 때문이다.
- Search API가 아직 `question_type` 같은 확장 필드를 소비하지 않기 때문에, 실제 외부 호출은 스펙 호환성을 우선하는 보수적 payload로 유지해야 한다.
- `Insufficient context` retry는 현재 1회만 수행하며, 초기 검색 결과가 완전히 비어 있는 경우보다 answer generation 이후 부족 판정 케이스에서 더 안정적으로 동작한다.
