- `Docling` 설치는 `torch` 및 모델 의존성 때문에 다운로드 용량과 설치 시간이 컸다.
- `Docling` 첫 실행 시 모델 다운로드가 발생한다.
- `DOC`는 `Docling` 대상이 아니므로 현재 구조상 `antiword` fallback이 맞다.
- Excel 샘플에서 `openpyxl` 기본 스타일 경고가 한 번 출력됐지만 파싱 결과 자체는 정상 생성됐다.
- PDF에 대해서는 `Docling`과 `PyMuPDF` 품질/속도 비교를 아직 끝내지 못했다.
- 화면 테스트와 브라우저 확인은 RAG 서버 기준으로만 진행한다.
- 현재 서버에는 소스만 남기고, 실제 앱 실행과 화면 확인은 RAG 서버에서만 유지한다.
- 장애 확인 순서는 `backend/logs/app.log` -> `run-logs/backend.out` -> `run-logs/frontend.out` 순서가 기준이다.
- 같은 `original_name` 문서를 여러 번 업로드하면 `Uploaded file list`에는 서로 다른 `stored_name` 항목으로 누적된다.
- 현재 구조에서 `parse` 메타데이터는 `stored_name`별 최신 parse 결과를 기준으로 저장되므로, 과거 실패 후 성공 재시도 시 화면에는 최신 성공 parse 결과가 보인다.
- parse 성공/실패 이력을 같은 행에서 함께 보여주는 구조는 작업을 진행했지만, RAG 서버 반영 결과는 다음 세션에서 다시 검증할 필요가 있다.
