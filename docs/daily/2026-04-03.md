## 3. 이슈 및 문제
- `Docling(md)`는 실제로 parse 후 Markdown을 만드는 모드이므로, “md만 만들고 파이프라인은 타지 않음” 동작은 아니다.
- PDF 운영 기본값은 여전히 `Legacy auto / PyMuPDF 우선`이 더 안전하다.
- `Docling`과 `Docling(md)`가 retrieval/answer 품질에 주는 차이는 아직 별도 비교 기록이 필요하다.
- upload list parse success/failure history UI는 구조상 남아 있지만 최종 화면 검증은 추가 확인이 필요하다.
- `/chat` external endpoint 경로는 UI와 backend 분기만 정리된 상태이고, 외부 RAG 실제 연동 검증은 아직 별도 대상 API가 필요하다.
- `/chat` frontend는 아직 `conversation_context` / `metadata` 입력과 rewrite 결과 표시를 붙이지 않았다.
