## 3. 이슈 및 문제
- 현재 frontend `/chat` 화면은 여전히 단일 질문 입력 중심이라 `conversation_context`를 직접 입력하는 UI는 없다.
- `Step 3` Standalone Search Query 검증은 아직 backend에 별도 단계로 분리돼 있지 않다.
- `Question`에서 role prefix가 없는 일반 멀티라인 텍스트는 여전히 단일 질문 문자열로 처리된다.
- 핵심 키워드 검증은 현재 문자열 포함 여부 중심이라, 도메인 동의어 확장은 이후 Search API 평가 단계와 함께 더 보강할 수 있다.
- `retrieved_chunks`의 `score`는 현재 `rerank_score` 우선, 없으면 raw `score`, 그다음 `distance` 기반 환산값을 사용하므로 외부 API 확정 시 재검토가 필요하다.
