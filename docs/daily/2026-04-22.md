## 3. 이슈 및 문제
- 현재 `document_type`은 LLM 힌트만 단독으로 사용하지 않고, metadata / document_hint / question_type rule을 함께 사용해 보수적으로 정규화한다.
- 통계/수치형 질문은 현재 정책상 `statistics_table`로 좁히며, 실제 recall 영향은 RAG 서버 호출 로그로 추가 확인이 필요하다.
- 외부 Search API가 `hits`와 `results`를 함께 반환하는 경우 `hits`가 중간 후보일 수 있어, `/chat`은 이제 `results`를 최종 근거 리스트로 사용한다.
- 기존 `/chat` 외부 Search payload는 `request.top_k` 대신 내부 candidate size 성격의 값을 보내고 있었고, 이번 수정으로 스펙 정의와 실제 요청 값을 일치시켰다.
- 내부 retrieval hit는 원래 `document_name` / `header_path`를 별도 필드로 쓰지 않았기 때문에, answer prompt에서 같은 포맷을 유지하려면 internal/external 공통 표준화가 필요했다.
- 상품명 mismatch 필터는 오탐을 줄이기 위해 보수적으로 구현했고, 질의/문서 비교가 애매하면 context를 유지하도록 했다.
