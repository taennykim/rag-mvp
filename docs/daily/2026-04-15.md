## 3. 이슈 및 문제
- 브라우저 화면 기준 `LLM Question` 표시와 실제 사용자 입력 흐름은 사용자가 RAG 서버에서 추가 확인 예정이다.
- 보험 도메인별 구체화 규칙은 현재 종신보험/사망보장 축을 우선 보강했으며, 다른 상품/담보 케이스는 테스트 사례가 쌓이면 추가 보강이 필요하다.
- 임시 Search API contract는 아직 확정본이 아니므로, 정식 contract가 나오면 payload/response mapping을 다시 정리해야 한다.
- 현재 RAG 서버 runtime에서 확인된 Azure OpenAI chat deployment는 `gpt-4o`이며, `gpt-4o-mini` 선택은 해당 deployment가 실제로 존재해야 정상 동작한다.
- 치조골 이식/수술특약/판결 문의 테스트에서는 외부 Search API 호출 자체는 정상이나, `rewritten_query`가 `치조골 이식 수술 보험 약관 수술 인정 여부?`로 짧게 축약되어 판결/수술특약 지급 여부 축이 약하게 반영됐다.
- 치조골 이식 테스트의 상위 검색 결과는 직접 관련 문서보다 일반 약관 조항이 잡혔고, Answer는 근거 부족으로 응답했다.
