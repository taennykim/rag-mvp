## 3. 이슈 및 문제
- query rewrite 단계는 현재도 LLM 응답 품질에 따라 `JSON object not found.`가 발생할 수 있다.
- 다만 이제는 response preview 로그가 남고 timeout이 `15초`로 줄어, 원인 분석과 장애 체감 시간은 이전보다 낫다.
- Search API 실패 메시지와 query rewrite 실패가 한 요청 안에서 함께 보일 수 있으므로, 실제 병목은 backend log 기준으로 구분해야 한다.
- RAG 서버 루트에는 과거 오동기화로 생긴 stray 파일(`main.py`, `page.tsx`, `plan.md` 등)이 남아 있어 이후에도 실제 실행 경로 기준으로만 동기화해야 한다.
